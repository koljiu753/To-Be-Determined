## 一、产品定位 & 核心功能

### 1.1 一句话定位

**TBD 是一款「工具内核 + 游戏皮肤」的 AI 决策推演产品** —— 用户输入真实生活中的 N 难抉择，AI 作为推演引擎，生成可视化的决策树，模拟每条路径可能导向的短/中/长期结果，帮助用户看清选择背后的代价与可能性。

### 1.2 核心价值主张

| 传统做法 | TBD 做法 |
|---------|----------|
| 自己想破头、列 Pro/Con 清单 | AI 结构化拆解变量，挖出你没想到的维度 |
| 找朋友咨询，受限于对方经验 | 调用大模型对普遍情况的认知，博采众长 |
| 心理咨询贵且慢 | 即时、低门槛、无评判 |
| 冷冰冰的分析报告 | 游戏化体验，像在玩自己人生的分支剧情 |

### 1.3 目标用户

**全年龄泛人群**，任何面临抉择的人。典型使用场景：

- 🎓 学生党：选专业、要不要考研、出国还是留下
- 💼 职场人：跳槽 vs 留下、接 offer A 还是 B、要不要创业
- 💑 情感：分手 or 继续、异地 or 同城、要不要结婚
- 🏠 人生大事：买房 vs 租房、移民、要不要孩子、转行
- 🤔 日常小抉择：周末出游选哪个目的地、投资选哪个标的

### 1.4 核心功能模块

#### 🎯 模块 A：困境输入 & 澄清（Clarify Agent）
- 自由文本输入当前困境
- AI 主动追问 3-5 个关键变量（年龄、经济状况、家庭、风险偏好、真实诉求等）
- 支持"快速模式"（一键补全假设）和"深度模式"（逐题澄清）

#### 🌳 模块 B：决策树生成（Simulation Agent）
- 基于澄清信息，生成 2-5 条主要路径
- 每条路径包含：
  - **路径标签**（例：稳妥派 / 冒险派 / 折中派）
  - **概率预估**（基于普遍情况的大致成功率）
  - **时间线推演**：3 个月 / 1 年 / 3 年 / 5 年的可能状态
  - **关键代价**（需要放弃什么）
  - **隐藏风险**（用户大概率没想到的点）

#### 🎮 模块 C：路径续推（Continuation Agent）
- 用户选中某条路径后，可以继续向下推演
- 模拟"如果这条路走到一半又遇到新问题"的二阶决策
- 像互动小说一样无限分支

#### 🗺️ 模块 D：决策地图可视化
- 树状图呈现所有路径
- 节点可展开查看详细推演
- 支持节点标注（用户自己的想法、补充信息）
- 可导出为图片/PDF 分享

#### 👤 模块 E：角色属性系统（游戏化皮肤）
- 用户可设置"人物卡"：性格标签、价值观排序、风险偏好
- AI 推演时会参考人物卡，生成更贴合用户的结果
- 角色属性影响路径结果的"适配度"评分

#### 📚 模块 F：历史 & 回访
- 保存所有决策会话
- 支持"未来回访"：3 个月后系统提醒，让用户反馈实际走向
- 沉淀个人决策数据，长期形成"决策画像"

### 1.5 产品原则

1. **不替用户做决定** —— AI 只呈现可能性，最终选择权在用户手里
2. **说人话** —— 避免装腔作势的"专业建议"，像一个睿智的朋友
3. **承认不确定** —— 明确标注推演的置信度，不给虚假的确定感
4. **情感安全** —— 面对沉重困境不冷漠，面对轻松话题不严肃

## 二、竞品分析

### 2.1 直接竞品

| 产品 | 定位 | 核心能力 | 缺口（TBD 机会点） |
|------|------|---------|-------------------|
| **GoalSim** | AI 人生目标模拟游戏 | 输入目标 → AI 叙事式推演 | 过于游戏化，对真实决策帮助有限；英文为主 |
| **Infinite Life Simulator** | AI 生活模拟器 | 生成式虚拟人生 | 娱乐属性强，不解决真实困境 |
| **Proxi**（Will Wright） | 基于记忆的 AI 模拟 | 镜像用户性格 | 研发阶段，偏学术，非决策工具 |
| **AI Dungeon** | AI 文字冒险 | 无限剧情生成 | 纯娱乐，无结构化决策输出 |

### 2.2 间接竞品

| 品类 | 代表产品 | 优势 | TBD 差异化 |
|------|---------|------|------------|
| **AI 心理咨询** | Replika、Woebot、小冰 | 情感陪伴 | TBD 更聚焦具体决策而非情绪疏导 |
| **通用 AI 助手** | ChatGPT、Kimi、豆包 | 强大通用能力 | 无结构化推演界面，输出不可视化，无记忆沉淀 |
| **决策类工具** | Notion 模板、FigJam | 结构化思维 | 纯工具无 AI，需要用户自己填内容 |
| **塔罗/MBTI 类** | 各类占卜/性格测试 App | 低门槛娱乐 | 伪科学；TBD 基于真实世界数据推演 |

### 2.3 市场空白

**中文市场几乎没有产品同时满足以下三点：**

1. ✅ 基于大模型的**结构化决策推演**（不是聊天）
2. ✅ **游戏化可视化**的交互体验（不是报告）
3. ✅ 聚焦**真实人生抉择**（不是娱乐）

这个交叉区域是 TBD 的护城河。

### 3.2 技术选型（不锁死在 Claude+GLM）

#### 前端
- **Next.js 14 (App Router)**：SSR + 良好 SEO + 部署简单
- **Tailwind CSS + shadcn/ui**：快速构建高质量 UI
- **React Flow / D3.js**：决策树可视化的核心库
- **Framer Motion**：过渡动画，增强游戏感
- **Zustand**：轻量状态管理

#### 后端
- **推荐 Python (FastAPI)**：AI 生态成熟，LangChain/LangGraph 原生支持
- **备选 Node.js (Hono/Nest)**：与前端同构，团队如果前端背景强可选
- **Celery / BullMQ**：异步任务队列（长推演任务）

#### 数据层
- **PostgreSQL**（主库）：用户、会话、决策树数据
- **Redis**：缓存 + 会话状态
- **Pinecone / Milvus / pgvector**：向量库，用于历史决策检索、相似案例召回
- **S3 兼容对象存储**（阿里云 OSS / Cloudflare R2）：用户导出文件

#### AI 层（重点）

**采用多模型路由策略**，不绑定单一供应商：

| 任务类型 | 推荐模型 | 备选 | 考量 |
|---------|---------|------|------|
| 澄清追问（轻量快速） | GLM-4-Flash / Qwen-Turbo | DeepSeek-V3 | 低延迟、低成本 |
| 核心推演（深度推理） | Claude Sonnet 4.6 | GPT-4o、Gemini 2.5 Pro | 推理链严密 |
| 中文情感语境 | GLM-4 / 豆包 Pro | Qwen-Max | 中文文化理解好 |
| 创意分支生成 | Claude Opus 4.7 | GPT-4.5 | 发散能力强 |
| 结构化输出（JSON） | 支持 function call 的模型 | — | 降低解析出错率 |

**路由策略示例：**

```python
# 伪代码
def route_model(task_type, user_tier):
    if task_type == "clarify":
        return "glm-4-flash"  # 便宜快速
    elif task_type == "simulate" and user_tier == "pro":
        return "claude-sonnet-4-6"  # 付费用户用好模型
    elif task_type == "simulate":
        return "deepseek-v3"  # 免费用户用平替
    elif task_type == "emotional":
        return "glm-4"
```

#### Agent 编排框架
- **LangGraph**（首选）：状态机式编排，适合多 Agent 协同
- **备选：LlamaIndex Workflows / AutoGen**

#### 部署
- **Vercel**：前端
- **Railway / Render / 阿里云 ECS**：后端
- **Cloudflare**：CDN + DDoS 防护

### 3.3 安全与合规

- 用户输入的都是隐私困境，**端到端加密存储**
- 敏感信息（具体姓名、公司名）入库前**自动脱敏**
- 遵循《生成式 AI 服务管理办法》，输出加免责声明
- 未成年人模式：检测到低龄用户时切换到更温和的推演

## 四、Agent 开发计划

### 4.1 Agent 架构图

```
                用户输入
                   │
                   ▼
         ┌──────────────────┐
         │  🎯 Triage Agent │  分流判断：是简单问题还是深度决策？
         └────────┬─────────┘
                  │
         ┌────────┴─────────────┐
         │                      │
         ▼                      ▼
 ┌──────────────┐      ┌────────────────┐
 │ 💬 Clarify   │      │  ⚡ Quick Answer│
 │   Agent      │      │  （轻量问题）    │
 └──────┬───────┘      └────────────────┘
        │
        ▼
 ┌──────────────┐
 │ 🌳 Simulator │  核心：生成多路径推演
 │   Agent      │  
 └──────┬───────┘
        │
        ▼
 ┌──────────────┐
 │ 🔍 Critic    │  自检：推演是否合理？有无遗漏？
 │   Agent      │
 └──────┬───────┘
        │
        ▼
 ┌──────────────┐
 │ 🎨 Render    │  生成结构化 JSON → 前端渲染决策树
 │   Agent      │
 └──────────────┘
```

### 4.2 五大核心 Agent 详解

#### Agent 1：Triage（分流 Agent）
- **职责**：判断用户输入属于"闲聊/简单问题/决策困境"
- **输出**：`{type: "decision" | "quick_q" | "chat", confidence: 0.9}`
- **关键点**：避免把所有输入都走重流程，节省成本

#### Agent 2：Clarify（澄清 Agent）
- **职责**：针对困境，识别信息盲区，主动追问
- **关键变量清单**（示例，实际覆盖 15-20 维）：
  - 时间紧迫度
  - 经济约束
  - 家庭/关系影响
  - 风险承受力
  - 用户真正在乎什么（often 隐含）
- **输出**：结构化用户画像 JSON

#### Agent 3：Simulator（推演 Agent）⭐ 核心
- **职责**：生成 2-5 条决策路径，每条包含多时间尺度结果
- **技术要点**：
  - 使用 CoT（思维链）提示
  - 调用「自我反思」机制：先给出初稿，再批判、再修订
  - 强制结构化输出（JSON Schema）
- **输出**：完整决策树 JSON

#### Agent 4：Critic（自检 Agent）
- **职责**：检查推演质量
- **检查项**：
  - 是否存在幸存者偏差？
  - 是否忽略了显而易见的风险？
  - 概率估计是否过于离谱？
  - 文字是否过度乐观/悲观？
- **输出**：修订建议，反馈给 Simulator 迭代

#### Agent 5：Render（渲染 Agent）
- **职责**：把 JSON 翻译成用户友好的可视化数据
- **包括**：节点布局、颜色标签、图标、过渡文案

### 4.3 记忆系统

- **短期记忆**：当前会话上下文（Redis）
- **长期记忆**：用户决策历史、人物卡、价值观偏好（Postgres + 向量库）
- **跨会话召回**：当用户遇到类似困境时，检索历史决策 + 后续反馈数据

## 五、Prompt 设计框架

### 5.1 系统提示词分层

```
┌──────────────────────────────────┐
│  L1: 人格设定（全局不变）          │
│  - 身份、语气、价值观、禁区         │
├──────────────────────────────────┤
│  L2: 角色指令（随 Agent 切换）     │
│  - Clarify 用什么风格追问          │
│  - Simulator 用什么结构输出        │
├──────────────────────────────────┤
│  L3: 动态上下文                   │
│  - 用户人物卡                     │
│  - 会话历史                       │
│  - 当前任务元信息                  │
└──────────────────────────────────┘
```

### 5.2 L1 人格设定模板

```markdown
你是 TBD，一个帮助用户梳理人生抉择的 AI 伙伴。

# 你的特质
- 睿智但不说教，像一个阅历丰富的朋友
- 诚实：会指出用户没想到的代价，不粉饰
- 克制：不替用户做决定，只呈现可能性
- 温度：面对沉重话题不冷漠，面对轻松话题不端着

# 你的工作方式
- 承认不确定：对任何预测都标注置信度
- 避免空话："可能会有风险"这种话是无效的
- 给具体例子：用"比如…"代替抽象描述
- 尊重用户：不假设用户需要被教育

# 绝对禁区
- 不提供医疗、法律、精神疾病的专业诊断
- 不替用户决定生育、离婚、自杀等重大不可逆选择
- 不基于种族、性别、地域做刻板推断
- 察觉到用户有自伤倾向时，优先提供求助资源
```

### 5.3 Simulator Agent Prompt（核心）

```markdown
# 任务
基于用户的困境和人物卡，生成 3-5 条决策路径的完整推演。

# 输出格式（严格 JSON）
{
  "paths": [
    {
      "id": "path_1",
      "label": "稳妥派：留在现公司",
      "tag": ["低风险", "慢增长"],
      "probability": 0.65,
      "timeline": {
        "3_months": "...",
        "1_year": "...",
        "3_years": "...",
        "5_years": "..."
      },
      "costs": ["放弃 X 的可能性", "错过 Y"],
      "hidden_risks": ["你可能没想到 Z"],
      "fit_score": 0.8  // 与用户人物卡的匹配度
    }
  ]
}

# 推演规则
1. **多元化**：路径之间必须有本质差异，不是同一选项的变体
2. **具象化**：每个时间点给出具体场景，不说"会变好/变坏"
3. **代价诚实**：每条路都有代价，"两全其美"的选项要加警示
4. **概率克制**：0.3-0.7 区间是常态，极端概率需要充分理由

# 思维链（内部执行，不输出）
Step 1: 分析用户的核心矛盾是什么？
Step 2: 列出所有可行选项（至少 5 个）
Step 3: 合并相似选项，保留 3-5 条差异化路径
Step 4: 对每条路径做时间线推演
Step 5: 批判性检查：有没有幸存者偏差？有没有忽略黑天鹅？
Step 6: 输出 JSON
```

### 5.4 Critic Agent Prompt

```markdown
你是一个严格的决策分析质检员。下面是 Simulator 生成的推演，请审查：

# 检查清单
- [ ] 每条路径是否真的差异化？（不是同一方案的变体）
- [ ] 概率估计是否合理？有无过度自信？
- [ ] 是否忽略了常见陷阱：幸存者偏差、沉没成本、乐观偏见？
- [ ] 时间线推演是否具体可信？
- [ ] 是否有"政治正确式"的空话？

# 输出
{
  "quality_score": 0-10,
  "issues": ["..."],
  "suggestions": ["..."]
}
```

### 5.5 Prompt 迭代流程

1. **原型期**：人工在 Claude/GPT 对话窗测试 20+ 真实场景
2. **评估期**：建立 Eval 集合（困境-理想推演对），用 LLM-as-Judge 打分
3. **生产期**：A/B 测试不同 Prompt 版本，按用户反馈优化

## 六、开发计划 & 里程碑

### 6.1 总览

| 阶段 | 时长 | 目标 | 产出 |
|------|------|------|------|
| **Phase 0: 验证** | 2 周 | 验证核心假设 | 纯 Prompt 原型 + 20 用户测试 |
| **Phase 1: MVP** | 8 周 | 跑通基础流程 | Web 可用版本 |
| **Phase 2: 游戏化** | 6 周 | 建立差异化体验 | 角色系统 + 可视化 |
| **Phase 3: 增长** | 持续 | 数据沉淀 & 社区 | 移动端 + 社交功能 |

### 6.2 Phase 0：验证（Week 1-2）

**核心假设：** 大模型能在真实困境上产出高质量推演。

- [ ] 用 Claude/GLM 在对话窗搭建 Prompt 原型
- [ ] 收集 20 个真实困境（自己的、朋友的、网络匿名分享的）
- [ ] 手动跑推演，邀请 5-10 人评分（有用度 1-5）
- [ ] **Go/No-Go 判断**：如果平均分 < 3.5，重新设计 Prompt 或放弃

### 6.3 Phase 1：MVP（Week 3-10）

**目标：** 能 demo 给陌生人用的 Web 版本。

#### Week 3-4：架构搭建
- [ ] 技术栈初始化（Next.js + FastAPI + Postgres）
- [ ] 模型网关层（支持至少 2 个模型供应商）
- [ ] 用户注册/登录（邮箱 + OAuth）

#### Week 5-6：核心 Agent
- [ ] Clarify Agent 实现（流式输出）
- [ ] Simulator Agent 实现（结构化 JSON）
- [ ] Critic Agent 实现（自检循环）

#### Week 7-8：前端
- [ ] 对话式输入界面

## 七、商业化路径

### 7.1 收费模型

**Freemium 订阅制**（主推）

| 版本 | 价格 | 权益 |
|------|------|------|
| **Free** | ¥0 | 每月 3 次深度推演、基础模型、无历史保存 |
| **Pro** | ¥29/月 或 ¥298/年 | 无限推演、旗舰模型、完整历史、导出、人物卡高级功能 |
| **Premium** | ¥68/月 | Pro 全部 + 优先推演队列 + 月度决策复盘报告 |

### 7.2 潜在变现路径

1. **订阅（核心）**：预估 3-5% 付费率是合理目标
2. **定制化模板**：垂直场景包（留学决策包、创业决策包等），¥19-49 一次性购买
3. **B 端合作**：
   - 生涯规划机构白牌合作
   - 心理咨询 App 作为辅助工具集成
   - 企业 EAP（员工援助计划）定制版
4. **数据洞察报告**（合规前提下）：匿名聚合数据生成"中国 Z 世代决策白皮书"等
5. **IP 衍生**：热门决策故事出版/剧集化（远期）

### 7.3 不该做的商业化

- ❌ 卖用户数据 —— 毁掉信任基石
- ❌ 植入广告 —— 破坏决策严肃性
- ❌ 付费解锁"正确答案" —— 与产品哲学冲突

### 7.4 增长策略

**冷启动阶段（0-1 万用户）**
- 小红书/知乎种草："AI 帮我决定要不要辞职"类内容
- 播客合作（聚焦职场、心理、成长类）
- 高校社团合作（考研季、求职季）

**成长阶段（1-10 万用户）**
- 用户生成内容（UGC）：决策故事分享
- KOL 合作测评
- SEO 内容：围绕"选择困难症""人生抉择"等长尾词

**成熟阶段（10 万+）**
- 品牌 campaign：年度"决策地图"总结（类似网易云年度报告）
- 跨界联名：心理学博主、成长导师

## 八、风险与应对

### 8.1 技术风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 模型幻觉导致错误推演 | 🔴 高 | Critic Agent 自检 + 用户标注反馈 |
| API 成本失控 | 🟡 中 | 分层模型路由 + Redis 缓存 + 限流 |
| 模型供应商政策变动 | 🟡 中 | 多供应商冗余，可快速切换 |
| 流式输出稳定性 | 🟢 低 | 降级方案：流式失败→ 非流式 |

### 8.2 产品风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 用户觉得推演"不准" | 🔴 高 | 强调概率性、非决定论；改进 Prompt |
| 沉重决策带来负面情绪 | 🔴 高 | 情感守护模块，必要时跳转求助资源 |
| 用户过度依赖 AI | 🟡 中 | 产品文案反复强调"最终你决定" |
| 隐私担忧 | 🟡 中 | 明确隐私政策，支持本地会话模式 |

### 8.3 合规风险

| 风险 | 应对 |
|------|------|
| 生成式 AI 监管 | 完成算法备案、内容安全审核接入 |
| 涉及未成年人 | 注册年龄门槛 + 未成年人模式 |
| 医疗/法律等专业建议红线 | 系统提示词严格禁区 + 输出过滤 |
| 跨境数据传输 | 如用海外模型，做好数据合规处理 |

### 8.4 伦理风险

- **避免制造"算命感"**：明确这是辅助思考工具，不是预言机
- **防止情绪操纵**：不利用用户焦虑做付费转化
- **尊重多元价值**：不预设"成功人生"的单一模板
- **留出人性空间**：AI 永远不做最终决定
