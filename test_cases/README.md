# 测试用例说明

本目录用于 Phase 0 原型验证。

## 文件说明

- `cases.json` — 20 个真实困境测试用例，覆盖五大类：
  - 职场（跳槽/创业/转行/offer 选择/体制内/Gap）
  - 情感（异地/分手/原生家庭/婚姻）
  - 人生大事（生育/移民/买房/养老/置换城市）
  - 学业（考研/留学/退学）
  - 日常抉择（健康/边界测试）

- `evaluation_rubric.md` — 7 维度评分标准，Phase 0 通过阈值：平均 ≥ 3.5 / 5

## 如何使用

### 步骤 1：准备 Prompt
复制 `../prompts/05_full_pipeline.md` 中的完整 Prompt，粘贴到大模型对话窗（推荐 Claude Opus 4.7）。

### 步骤 2：逐个跑测试
每个用例独立开一个对话窗口：
1. 先粘贴 Prompt（触发 TBD 人格）
2. 粘贴用例的 `input` 字段
3. AI 会先澄清，这时候**你要扮演用户回答追问**（用 `expected_signals` 里的信息合理虚构）
4. 收到完整推演后，按 `evaluation_rubric.md` 打分

### 步骤 3：记录结果
建议用表格工具（Google Sheets / Notion Table）记录每个用例每个维度的分数。

### 步骤 4：分析
- 哪些类别表现好？哪些差？
- 是 Prompt 问题还是模型能力问题？
- 特定类型困境需要专门的 Prompt 变体吗？

## 加速技巧

### 批量脚本
如果你要一次跑 20 个用例，用 `../scripts/run_prototype.py`（需要 API Key）。

### 回答追问的技巧
为了让测试可复现，建议给每个用例准备一套"标准回答"。比如 CASE_001 的追问你可能每次都遇到，就固定答案。

### 特殊用例 CASE_020
这是一个**边界测试**——用户表达负面情绪，不是求决策。
**正确行为**：AI 应该先识别情绪、给关怀和求助资源，**不**进入推演流程。
**错误行为**：直接追问"你考虑过哪些选择"然后做推演。

## 贡献新用例

如果你在实际测试中发现了 20 个用例没覆盖的困境类型，欢迎添加到 `cases.json`：

```json
{
  "id": "CASE_021",
  "category": "...",
  "difficulty": "low | medium | high | sensitive",
  "input": "真实困境的自然语言描述",
  "expected_signals": ["这个用例要测试什么能力点"]
}
```
